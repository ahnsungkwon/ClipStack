# ClipStack을 실제 작업 흐름에 넣어보니 달라진 점

안녕하세요.
지난 글에서는 왜 ClipStack을 만들게 되었는지, 그리고 어떤 불편을 줄이기 위해 시작했는지에 대해 정리해봤습니다.
이번 글에서는 조금 더 실사용 관점에서,
**실제 작업 흐름에 넣었을 때 뭐가 달라졌는지**를 정리해보려고 합니다.

---

## 1. 예전에는 무엇이 불편했나
반복 작업이 많은 날에는 복사 붙여넣기 자체보다,
이전에 복사했던 내용을 다시 찾는 순간이 더 피곤했습니다.

예를 들면 이런 것들입니다.
- 담당자명
- 날짜
- 사번
- 이벤트명
- 자주 쓰는 설명 문구
- 제목/본문/해시태그 조합
- 프롬프트 문구
- 로그 확인용 명령어

이런 텍스트는 자주 쓰지만 완전히 똑같은 형태로만 반복되지는 않습니다.
그래서 메모장이나 기존 클립보드 히스토리만으로는,
다시 찾고 다시 쓰는 과정이 은근히 자주 흐름을 끊었습니다.

---

## 2. ClipStack을 넣고 달라진 점
ClipStack을 넣고 가장 달라진 점은
`많이 저장하게 된 것`이 아니라,
**자주 쓰는 묶음을 다시 꺼내 쓰는 시간이 줄었다는 점**입니다.

예전에는
- 메모장을 열고
- 필요한 줄을 찾고
- 하나씩 복사하고
- 다시 입력 화면으로 돌아가야 했습니다

지금은
- 자주 쓰는 3개 항목을 세트로 저장하고
- 필요할 때 다시 주입하고
- 세트만 바꿔가며 쓰는 식으로 정리됩니다

이 차이가 아주 거창해 보이지는 않지만,
반복이 쌓이면 체감은 꽤 큽니다.

---

## 3. 특히 편했던 순간들
### 1) 행정/등록 업무
담당자, 부서, 내선처럼 항상 같이 움직이는 값들을 세트로 묶어두면,
이벤트마다 다시 정리하는 시간이 줄어듭니다.

### 2) 콘텐츠 업로드
제목, 짧은 설명, 해시태그처럼 같이 쓰는 항목들을 세트로 관리하면,
여러 버전을 번갈아 쓸 때 편합니다.

### 3) 개발/문서 작업
프롬프트, 로그 확인 명령어, 자주 쓰는 템플릿 문구를 다시 꺼내 쓰는 흐름이 단순해집니다.

---

## 4. 아직 부족한 점도 있습니다
물론 아직 완벽한 도구는 아닙니다.
작고 가벼운 유틸리티로 시작한 만큼,
실제 사용 피드백을 받으면서 더 다듬어야 할 부분이 있습니다.

예를 들면,
- 어떤 단축키 구성이 가장 자연스러운지
- 세트 수가 지금보다 더 필요한지
- 관리창 UX를 더 단순하게 줄일 수 있는지
- 어떤 작업군에서 가장 체감이 큰지
같은 부분은 더 확인해보고 싶습니다.

---

## 5. 그래서 지금은 피드백이 가장 중요합니다
지금 단계에서 ClipStack은 설치 수를 무조건 늘리는 것보다,
**어떤 작업에서 실제로 쓸 만한지 반응을 모으는 것**이 더 중요하다고 생각합니다.

README 첫부분에는 왜 만들었는지와 GIF 데모를 같이 넣어두었습니다.
직접 보시고,
- 어떤 흐름에서 특히 편해 보이는지
- 어디가 아직 애매한지
- 어떤 기능이 있으면 더 좋을지
의견을 주시면 정말 도움이 됩니다.

---

## 6. 마무리
ClipStack은 클립보드를 많이 저장하는 도구라기보다,
반복 작업에서 다시 찾고 다시 쓰는 흐름을 덜 끊기게 만들기 위한 도구입니다.

작은 차이 같지만,
비슷한 텍스트를 자주 다루는 사람에게는 꽤 체감이 될 수 있다고 생각합니다.

혹시 비슷한 작업을 자주 하신다면,
README 첫부분 데모를 한번 보시고 의견 남겨주셔도 좋겠습니다.
