# ClipStack Launch Plan

## Goal
Turn ClipStack from a useful personal utility into a tool that immediately feels relatable to anyone doing repetitive copy/paste work.

## Core message
ClipStack is not just a clipboard tool.
It solves a very specific pain:
- repeating the same registration/input work,
- switching between slightly different text groups,
- and wasting time copying items one by one.

## Positioning
- **Before**: Users copy one line at a time and constantly reopen clipboard history.
- **Discovery**: Windows already has Win+V multi-clipboard, which is great.
- **Problem left unsolved**: real work needs reusable grouped sets, not only recent history.
- **After**: ClipStack adds reusable 3-item sets, fast switching, and practical workflow control.

## Target users
1. Online registration / data entry workers
2. Office admins handling repeated forms
3. Sellers and operators entering repeated product/event info
4. Students and researchers reusing grouped notes/snippets
5. Anyone who lives in copy/paste-heavy workflows

## Content pillars
1. **Pain → Solution**
   - "복붙 너무 많이 하는 사람을 위한 도구"
2. **Workflow demo**
   - Show Grab → Stack → Sync → Switch visually
3. **Windows hidden feature + upgrade story**
   - Win+V is the discovery, ClipStack is the upgrade
4. **Real productivity angle**
   - less friction, fewer mistakes, faster switching

## Priority channels
### 1) YouTube Shorts / Reels
Best for showing the workflow quickly.
- 15 to 20 sec is enough
- hook in first 2 to 3 sec
- start from pain, not features
- show the workflow immediately after the pain hook
- use the existing GIF/video as the visual backbone

Recommended title example:
- `윈도우에서 아직도 Ctrl+C, Ctrl+V 하나씩 하세요? (생산성 꿀팁)`

### 2) Blog post
Best for search and explanation.
- story of discovery
- why Win+V helps but is not enough
- how ClipStack solves the next step
- install + usage guide + GIF/demo
- position it as `Win+V의 한계 극복` rather than a generic clipboard app

Recommended title example:
- `[무료/윈도우] 불편해서 직접 만든 클립보드 확장 프로그램 (Win+V의 한계 극복)`

### 3) Instagram / Threads style posts
Best for awareness and saves/shares.
- carousel: problem, discovery, limitation, solution, demo
- caption focused on empathy and workflow pain
- use short lines and practical screenshots

### 4) Community seeding
Best for trust, feedback, and first users.
- Clien tips/news style boards
- Ppomppu computer/dev boards
- AutoHotkey / Windows productivity communities
- Open chat rooms for productivity, office workflow, or dev tools

Tone rule:
- do not sound like an ad
- sound like someone sharing a genuinely useful tool they built for a real annoyance

## 7-day rollout suggestion
### Day 1
- README/GIF/screenshots already prepared
- Publish blog article first
- Upload Shorts #1: `Win+V 알았을 때의 충격`
- Post Instagram/Reels version

### Day 2
- Share community post to 2 or 3 communities
- Use a soft intro, not a promotional tone

### Day 3
- Upload Shorts #2: `왜 기본 멀티클립보드만으로는 부족한가`

### Day 4
- Post carousel: `반복 입력 업무를 세트로 바꾸는 법`

### Day 5
- Upload Shorts #3: hotkey demo (`Ctrl+Alt+1~6`)

### Day 6
- repost best-performing short to another platform
- collect comments / user pain phrases

### Day 7
- recap post with repo + best demo clip + first feedback

## 3-day practical action guide
### Today
- publish the blog post on Naver blog or Tistory
- use the story angle, not a feature list first

### Tomorrow
- share to 2 to 3 communities with GIF and GitHub link

### The day after
- create one polished 15-second short in CapCut or Vrew and upload to YouTube + Instagram

## Conversion path
1. Short video catches attention
2. Blog/README explains the story
3. GitHub gives proof and download path
4. Users try it in real work

## Success metrics
- GitHub repo visits / stars
- README GIF engagement
- Shorts views / retention
- Saves / shares on Instagram
- Comments like "이거 나한테 필요했다"

## Important tone
Do not market it like a generic productivity app.
Market it like:
- "I made this because repetitive real work was annoying"
- "Windows already had part of the answer"
- "ClipStack completes the workflow"
- "This is a small but practical tool for people doing real repetitive work"
- "Share usefulness first, promote second"
