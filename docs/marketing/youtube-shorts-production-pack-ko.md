## Shorts 1. 문제 발견형

### 목적
Win+V를 모르거나, 기본 클립보드만 쓰는 사람에게 문제와 해결 흐름을 한 번에 이해시키기.

### 제목 후보
- 복붙 많이 하는 사람은 왜 항상 흐름이 끊길까?
- 윈도우에서 아직도 Ctrl+C, Ctrl+V 하나씩 하세요?
- Win+V 알았을 때도 끝이 아니었습니다

### 영상 길이
22~28초

### 자막 완성본
복붙 많이 하면
일이 빨라지는 게 아니라
오히려 흐름이 끊길 때가 있습니다

Win+V를 알면 조금 편해지지만
실무에서는 그것만으로 부족할 때가 있어요

ClipStack은
클립을 많이 저장하는 도구보다
다시 꺼내 쓰는 과정을 줄이려고 만들었습니다

README에 GIF 데모 올려뒀어요
보고 의견 주세요

### 나레이션 버전
복붙이 많은 작업은 생각보다 자주 흐름을 끊습니다.
Win+V를 알면 분명 편해지지만, 실제 업무에서는 거기서도 부족할 때가 있어요.
ClipStack은 클립을 그냥 쌓는 도구가 아니라, 다시 꺼내 쓰는 과정을 줄이려고 만들었습니다.
README에 데모 올려뒀으니 보고 의견 주세요.

### 컷 편집표
| 시간 | 화면 | 자막/연출 |
|---|---|---|
| 0~2초 | 엑셀/폼/메모장에서 한 줄씩 복붙하는 장면 | "복붙 많이 하면 흐름이 끊깁니다" |
| 3~6초 | Win+V 패널 열기 | "Win+V를 알면 조금 편해지지만" |
| 7~11초 | 복사 항목이 많아져 다시 찾는 장면 | "막상 다시 찾으려면 또 귀찮죠" |
| 12~18초 | ClipStack 세트 선택 + 주입 장면 | "다시 꺼내 쓰는 과정을 줄였습니다" |
| 19~24초 | 세트 전환 장면 | "세트 저장 + 빠른 전환" |
| 25~28초 | README/GitHub 주소 화면 | "README 데모 보고 의견 주세요" |

### 설명란 초안
복붙이 많은 작업에서 제일 불편한 건 복사 자체보다,
예전에 복사한 내용을 다시 찾는 순간이었습니다.

ClipStack은 저장 기능보다
다시 꺼내 쓰는 흐름을 줄이는 쪽에 집중한 작은 도구입니다.

GitHub:
https://github.com/ahnsungkwon/ClipStack

## Shorts 2. 실무 공감형

### 목적
온라인 등록, 행정, 운영, 반복 입력 업무를 하는 사람에게 강하게 공감시키기.

### 제목 후보
- 온라인 등록 업무 하루 10번 하는 사람만 보세요
- 담당자명, 날짜, 사번 복붙이 많다면
- 반복 입력 업무를 세트로 바꾸면 편해집니다

### 영상 길이
20~30초

### 자막 완성본
날짜
담당자
사번
이런 거 매번 한 줄씩 복붙하고 있나요?

문제는 복사가 아니라
다시 찾고 다시 붙이는 흐름입니다

ClipStack은
자주 쓰는 텍스트를 세트로 저장하고
바로 다시 불러올 수 있게 만들었습니다

이런 작업에도 쓸 만한지
피드백 주세요

### 나레이션 버전
날짜, 담당자, 사번처럼 조금씩 바뀌는 내용을 반복해서 넣는 작업은 생각보다 피로합니다.
문제는 복사 그 자체보다, 다시 찾고 다시 붙이는 흐름이 계속 끊긴다는 점이었어요.
ClipStack은 자주 쓰는 텍스트를 세트로 저장하고 바로 다시 불러올 수 있게 만든 도구입니다.
이런 작업에도 쓸 만한지 피드백 주세요.

### 컷 편집표
| 시간 | 화면 | 자막/연출 |
|---|---|---|
| 0~3초 | 메모장에 담당자명/날짜/사번 적어둔 장면 | "이거 하루에 몇 번 하세요?" |
| 4~8초 | 하나씩 복사해서 폼에 넣는 장면 | 느린 속도, 답답한 BGM |
| 9~14초 | ClipStack 관리창에서 3개 항목 세트 저장 | "세트로 저장" |
| 15~20초 | 단축키로 세트 불러와 Win+V/붙여넣기 | "바로 다시 사용" |
| 21~26초 | Before/After 비교 | "작업 흐름이 덜 끊깁니다" |
| 27~30초 | GitHub/README 화면 | "이런 작업에도 쓸 만한지 피드백 주세요" |

### 설명란 초안
반복 입력 업무에서 진짜 불편한 건
복사가 아니라 다시 찾고 다시 붙이는 흐름입니다.

ClipStack은 그 흐름을 덜 끊기게 만들려고 만든 도구예요.
README 첫부분에 GIF 데모도 넣어뒀습니다.

GitHub:
https://github.com/ahnsungkwon/ClipStack

## Shorts 3. 제작 스토리형

### 목적
기능 홍보보다 제작 배경과 진정성을 전달해서 공감/응원/피드백을 끌어내기.

### 제목 후보
- Windows 기본 기능 몰라서 직접 만든 프로그램
- 왜 또 클립보드 도구를 만들었는가
- 작은 불편을 줄이려고 만든 도구, ClipStack

### 영상 길이
18~24초

### 자막 완성본
처음엔 Win+V도 몰랐습니다

알고 나서는 분명 편해졌어요
근데 실무에서는 아직 부족했습니다

복사한 내용을 많이 쌓는 것보다
필요할 때 다시 꺼내 쓰는 게 더 중요했거든요

그래서 ClipStack을 만들었습니다

README에 왜 만들었는지도 적어뒀어요

### 나레이션 버전
처음엔 Win+V도 몰랐습니다.
알고 나서는 정말 편해졌어요.
그런데 실제 업무에서는 아직 부족했습니다.
많이 복사하는 것보다, 필요한 내용을 다시 빠르게 꺼내 쓰는 게 더 중요했거든요.
그래서 ClipStack을 만들었습니다. README에 왜 만들었는지도 적어뒀어요.

| 4~8초 | Win+V 보여주는 장면 | "알고 나서는 분명 편해졌어요" |
